Introducción y creación

IndexedDB

Es una manera de almacenar datos de manera persistente en el navegador. Almacena pares key-value (llave-valor). Los valores pueden ser objetos con estructuras complejas, y las llaves pueden ser propiedades de esos objetos.
La API de IndexedDB es mayormente asíncrona.
IndexedDB usa eventos DOM para notificar cunado los resultados están disponibles y es orientada a objetos.
NOTA:
La principal diferencia entre IndexedDB y LocalStorage es que, esta ultima sirve para porciones de datos pequeñas mientras que IndexedDB sirve para grandes cantidades de datos. IndexedDB es una base de datos indexada no sql

Creación

Crear la base de datos a través del objeto indexedDB() y el método open()
Comprobar si la base de datos existe o tiene que ser creada a través del método onupgradeneeded()
Crear almacén de objetos con el método createObjectStore()
Escuchar los eventos de éxito y de error con los métodos onsucces() y onerror()
NOTA:
.Cuando accedemos a una base de datos relacional o de tablas, si no existe no se puede accerder a ella. Con las bases de tados no-sql o no relacionales esto no pasa, si no existe al acceder a ella, el sistema la crea.
.Al lugar donde guardamos los datos en IndexedDB se le denomina "almacenes", en otras bases de datos no-sql como MongoDB y FireBase se les denomina "colecciones".
